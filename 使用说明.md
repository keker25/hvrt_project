## 🚀 快速开始（推荐）

### 方式一：运行完整演示（无网络依赖）⭐

直接运行这个脚本，它会模拟所有服务并展示完整认证流程：

```bash
python full_demo_no_network.py
```

**这个脚本包含：**
- ✅ Ed25519 密码学
- ✅ GTT 全局信任票据签发与验证
- ✅ RRT 区域注册票据
- ✅ SAT 会话访问票据
- ✅ HMAC-SHA256 挑战响应
- ✅ 完整票据链验证 (SAT→RRT→GTT)
- ✅ 设备注册、撤销、漫游
- ✅ 状态同步 (CTA→EC→AG)

### 方式二：运行独立演示（核心功能）

```bash
python standalone_demo.py
```

## 📦 安装完整依赖后运行真实服务

如果要运行真实的网络服务（CTA/EC/AG 分离部署），请先安装依赖：

```bash
pip install -r requirements.txt
```

如果有权限问题，尝试：

```bash
pip install --user -r requirements.txt
```

### 启动服务

在 **4 个不同的终端** 分别运行：

```bash
# 终端 1 - CTA (云端信任中心，端口 8000)
python -m cta.main

# 终端 2 - EC (边缘协调，端口 8050)
python -m ec.main

# 终端 3 - AG1 (接入网关 1，端口 8100)
python -m ag.main --port 8100

# 终端 4 - AG2 (接入网关 2，端口 8200)
python -m ag.main --port 8200
```

### 运行演示

在第 5 个终端运行：

```bash
run_demo.bat
```

## 🔐 核心特性

1. **分层信任架构**
   - GTT (全局) → RRT (区域) → SAT (会话)
   
2. **离线认证**
   - AG 可本地验证，无需 CTA 在线
   
3. **密码学安全**
   - Ed25519 数字签名
   - HMAC-SHA256 挑战响应
   
4. **状态同步**
   - CTA → EC → AG 增量同步撤销状态
   
5. **漫游支持**
   - TD 可在不同 AG 间无缝漫游
